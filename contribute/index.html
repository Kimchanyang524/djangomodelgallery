행동 강령

모든 참여자는 Contributor Covenant를 준수해야 합니다.

오픈소스

개발 작업 주로 GitHub에서 이루어집니다. 노션으로만 작성해서 링크만 첨부하셔도 좋습니다.

PR 규칙

변경 사항은 풀 리퀘스트를 통해 제출되며, 린트 및 테스트를 거칩니다.

유의적 버전

유의적 버전을 따르며, 변경 사항은 README.md 파일에 기록됩니다.

버그 보고

GitHub Issues를 통해 버그를 보고하고, 보안 관련 버그는 적절한 절차를 따릅니다.

변경 제안

변경 사항에 대한 제안은 이슈를 통해 논의되며, 코어 팀과 합의를 이뤄야 합니다.

CLA 동의

풀 리퀘스트를 제출하려면 Contributor License Agreement에 동의해야 합니다.

스타일 가이드

코드 스타일을 맞추기 위해 Prettier와 linter를 사용합니다.
